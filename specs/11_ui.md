# 模块说明书：页面交互层 (UI)

## 所属层级
页面交互层

## 目标目录
`app/`

生成文件：
```
app/
    __init__.py
    main.py                 # Streamlit 主入口
    pages/
        __init__.py
        upload_page.py      # 上传页
        task_page.py        # 任务页
        report_page.py      # 报告页
        benchmark_page.py   # Benchmark 页
```

## 依赖模块

- **必须先生成**：`07_application`（调用 SessionManager, TaskOrchestrator）
- **必须先生成**：`10_reporting`（展示 ReportBundle）
- **可选**：`12_evaluation`（Benchmark 页面）

## 职责边界

**做：**
- 提供文件上传界面（PDF/TXT/MD/CSV/Excel）
- 提供文本输入界面（用户研究问题/分析要求）
- 展示任务执行进度（步骤名、状态、工具调用日志）
- 展示最终报告（Markdown 渲染）
- 展示图表
- 提供报告下载入口
- 提供 Benchmark 跑分界面

**不做：**
- 不包含任何 LLM 调用
- 不直接访问 ChromaDB
- 不直接执行代码
- 不解析文件

## 关键交互流程

```
用户操作                    UI 变化                   后端调用
─────────────────────────────────────────────────────────────────
上传文件              -> 文件列表更新              保存到 data/uploads
                      -> "开始分析"按钮可用
输入问题 + 点击开始    -> 按钮变 loading            SessionManager.create_task()
                      -> 切换到任务看板            TaskOrchestrator.start_task()
任务执行中            -> 看板进度条刷新            轮询 get_task_status()
                      -> 日志流追加
任务完成              -> 自动切到报告页            get_task() -> report
                      -> 展示报告和图表
用户下载              -> 浏览器下载                 读取产物文件
```

## 实现约束

1. **Streamlit 特性适配**：
   - `st.session_state` 存储会话状态，不要用 Python 全局变量
   - 耗时操作不要用 `st.spinner` 包裹，用 `st.status` 展示进度
   - 使用 `st.rerun()` 实现自动刷新，不用 `while True`

2. **文件上传限制**：
   - 单文件上限 50MB，总上传上限 200MB
   - 上传前校验扩展名
   - 文件保存路径：`data/uploads/{session_id}/{filename}`

3. **响应式**：
   - 使用 `st.columns` 实现左右分栏
   - 图表展示区使用 `use_container_width=True`

4. **异常不崩溃**：
   - 任务失败时页面不能崩溃，显示错误提示
   - 所有 Streamlit 回调用 try/except 包裹

5. **空状态美化**：
   - 无任务时显示引导文字
   - 无报告时显示"请提交分析任务"

## 与上下游模块的对接

- **下游调用方**：无（最上层）
- **下游依赖方**：`application` 模块（SessionManager, TaskOrchestrator）
- **数据流向**：`用户 -> UI -> SessionManager/TaskOrchestrator -> UI 更新`

## 测试要点

- 上传文件后能在 `data/uploads` 中找到
- 点击开始后任务状态从 pending 变为 running
- 任务完成后报告页面能正确渲染 Markdown
- 页面刷新后（st.rerun）状态不丢失
- 空状态页面不报错
